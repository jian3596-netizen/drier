# 目录说明 — STM32 控制板固件

> 构建系统：**CMake + arm-none-eabi-gcc**（纯 VSCode，无 Keil）。旧 Keil 工程已删除。
> 编译/烧录步骤见 `README_编译环境.md`。

这个文件夹分两类：**源代码** 和 **构建系统文件**。

---

## 🟢 源代码（固件本体）

| 项 | 是什么 |
|---|---|
| `Core/` | CubeMX 生成的外设初始化（`gpio/adc/dma/tim/usart`）+ `main.c`（精简的主循环）+ 中断 `stm32f0xx_it.c` |
| `Drivers/` | ST 官方 HAL 库 + CMSIS（芯片底层，不用动）|
| `USER_DRIVE/` | 驱动层：`ADC_DRIVE`（采集 NTC 电阻+电压）、`DWIN_DRIVE`（给屏发数据）、`usart_drive`（串口缓冲）|
| `USER_APPLICATION/` | **`control.c/.h` — 控温的核心**：双区 PI 控温 + 安全联锁 + 看门狗 |
| `RTT/` | SEGGER RTT 调试打印 |
| `HGJ.ioc` | CubeMX 配置文件（**以后改引脚/外设时**用 CubeMX 打开它）|
| `.mxproject` | CubeMX 内部记录文件 |
| `日志.txt` | 交接时的开发进度日志 |

> **改控温逻辑 = 改 `USER_APPLICATION/control.c`**（PI 参数、安全阈值都在文件顶部的 `g_cfg` 配置结构体，可在 J-Link 调试器实时改）。
> `main.c` 现在很薄：初始化 + `while(1){ 采集 → 发屏 → Control_Update() }`。

## 🔵 CMake 构建系统（纯 VSCode）

| 项 | 是什么 |
|---|---|
| `CMakeLists.txt` | 工程定义（源文件/头路径/宏/编译链接选项）|
| `CMakePresets.json` | 一键 debug/release 配置 |
| `cmake/` | GCC 工具链定义文件 |
| `STM32F030F4Px_FLASH.ld` | 链接脚本（内存布局 16K Flash/4K RAM）|
| `.vscode/` | 扩展推荐 + J-Link 烧录调试配置 |
| `.gitignore` | 忽略编译产物（`build/`、`*.hex` 等）|
| `README_编译环境.md` | **怎么装、怎么编、怎么烧的步骤** |

> 这几个**必须**放在工程根目录（CMake 规矩：`CMakeLists.txt` 在源码树根部），和源码同级属正常。

## 本次重写做了什么（2026-06）
- 旧的开关式(bang-bang)控制 → **双区 PI 控温**（加热1=TIM16/PA6 跟 temp1，加热2=TIM1_CH2/PA9 跟 temp2）。
- 新增**安全联锁**：通信超时关加热、IWDG 看门狗、NTC 开路/短路检测、过温切断、加热时风扇强制开。
- 删除死代码：`HC595_DRIVE`、`HALL_DRIVE`、任务调度框架(`task`/`task_dispatch`)、`Temperature.c`（678 行，几乎全是注释表）、插入检测(微动开关 `KEY1/KEY2`)的 GPIO 残留。
- 删除整个旧 Keil 工程，驱动文件重写为干净 ASCII（解决 GBK 乱码）。
- 已编译/烧录/上机运行（**V2.0**）；PI 增益与安全阈值是占位值，仍需台架整定（集中在 `control.c` 的 `g_cfg` 结构体）。
- 后续(2026-06)：补回**加热片过温保护**（温3 ≥ `g_cfg.plate_temp_max`=128 → 切两路加热）；可调参数改为 `g_cfg` 配置结构体（调试器可实时改）；删插入检测残留。
